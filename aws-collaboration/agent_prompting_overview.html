Why Editors Should Care

These agents are not one generic reviewer

What the system is doing

  • Breaking screening into narrower editorial questions: scope, novelty, ethics, integrity, reporting, statistical verification.
  • Giving each sub-question a different prompt, a different tool set, and a different evidence base.
  • Forcing most agents to return structured fields rather than free-form opinions.
  • Passing outputs forward so later agents synthesize prior assessments instead of starting from scratch.

What editorial feedback can improve

  • Whether the right evidence sources are being treated as authoritative.
  • Whether the decision thresholds match real editorial practice.
  • Whether the prompts over-weight reporting completeness versus scientific merit.
  • Where transfer, reject, or further-investigation logic needs tighter policy language.

The key question is not "Is the model smart?" It is "Which instructions and which evidence sources are steering each editorial judgment?"

Workflow Map

End-to-end editorial flow

Gatekeepers

Step 2

Parallel analysis

Each agent gets its own prompt and resource layer.

Step 3

Synthesis

Step 4

Commentary

Shared Prompting Patterns

Five strategies repeated across the repo

1. Role-specialized prompts

Every agent is cast as a specific reviewer: "Scoping Specialist", "Guidelines Compliance Specialist", "editorial commentary writer", and so on.

2. Retrieval before judgment

Prompts explicitly name files, URLs, or search tools to consult before making the decision.

3. Structured outputs

Agents usually emit Pydantic-shaped decisions instead of unconstrained prose, which keeps downstream synthesis predictable.

4. Few-shot guidance where needed

Scoping and novelty append sampled examples from local JSON files to calibrate rejection and fit judgments.

5. Tool-constrained reasoning

The important distinction is not only the prompt text. It is prompt text plus tool access: `file_read`, gateway search, cached crawled pages, or Python execution.

Cross-cutting implementation habits

Most agents run with low temperature, prompt caching, and a clear evidence-first instruction style. Some add extended thinking for synthesis or validation.

Scoping And Novelty

Journal fit uses local scope packs; novelty uses live literature search

Scoping agent

  • Prompt placeholders are templated with a journal slug and display name before execution.
  • Decision prompt points the model to runtime journal packs under `journal_scopes/{journal}/scope_summary.txt` and `article_types.txt`.
  • Transfer prompt uses `journal_scopes/metadata.json` plus candidate journal scope packs.
  • Few-shot examples are appended from `backend/agents/scoping/prompts/few_shot_examples.json`.
  • Judgment style: topic fit + study type fit + transfer/reject fallback.
Use file_read to access: - journal_scopes/{journal}/scope_summary.txt - journal_scopes/{journal}/article_types.txt

Novelty agent

  • Prompt explicitly frames four editorial questions: how much literature exists, whether the topic is novel and important, whether the rationale is good, and what similar efforts exist.
  • Relies on MCP search tools for OpenAlex, PubMed, Google Scholar, general web search, and content extraction.
  • Uses search horizon rules: recent literature first, then expand if sparse, then landmark work.
  • Can append example novelty rejections from `backend/agents/novelty/prompts/few_shot_examples.json`.

Scoping prompt excerpt

You are a Scoping Specialist for {journal_name}.

Your task: Evaluate if a manuscript fits
{journal_name} based on BOTH topic
relevance AND study type.

Novelty prompt excerpt

Answer these 4 critical editor questions:
1. How much and what research already exists?
2. Is the topic novel, interesting and important?
3. Is there a good rationale?
4. What other research efforts already exist?
Ethics Stack

Ethics combines cached policy content, live retrieval, and policy-specific sub-agents

Primary compliance pass

  • The main ethics prompt injects named BMJ, UKRI/MRC, and NHS HRA URLs directly into the system prompt.
  • It asks the model to first try `get_url_content(url)` for cached local copies, then fall back to gateway web extraction.
  • It requires checklist extraction from each source before manuscript assessment.
  • The repo contains a local crawl index at `backend/agents/ethics/crawled_data/metadata.json` plus cached pages under `crawled_data/`.

Secondary ethics checks

  • Pre-specification agent reads a local `guidelines/declaration_of_helsinki.txt` via `file_read` and maps manuscript evidence to relevant articles.
  • Open science agent uses COS preregistration guidance and emphasizes date extraction plus timeline consistency.
  • Conflict-of-interest agent uses BMJ policies, ICMJE guidance, ORCID, PubMed, OpenAlex, and web extraction to compare declared versus discovered conflicts.
  • The workflow only runs these secondary checks after the first ethics pass is not low.

Methodology Reporting

Reporting review is a two-stage prompt chain with local EQUATOR and CASP assets

Stage 1: detect the right reporting guideline

  • `flowchart.yaml` stores a local EQUATOR decision tree.
  • The methodology agent asks one yes/no question at a time, records the decision path, and routes to a guideline endpoint such as CONSORT, PRISMA, STROBE, CARE, or TRIPOD.
  • If none of the standard categories fit, the prompt allows web search against the broader EQUATOR library.

Stage 2: parallel compliance assessment

  • Guidelines Specialist performs qualitative strengths, gaps, and methodology issues.
  • Checklist Auditor scores every checklist item and uses `calculate_score_from_table` for quantitative compliance.
  • CASP Analyzer chooses the right CASP checklist from local `casp_checklists/.../checklist.txt` files.
  • A methodology synthesizer merges the three views into one compliance report.

CASP packs `casp_checklists/randomised_control_trials`, `cohort_study`, `qualitative`, etc. Adds a methodological appraisal layer, not just a reporting layer.

Integrity And Validation

Integrity is search-heavy; validation is computation-heavy

Integrity agent

  • The prompt is explicit about the evidence sequence: PubPeer, ORCID, OpenAlex, PubMed, Google Scholar, then general web context.
  • It emphasizes author disambiguation using affiliation and topic area and instructs the model not to guess.
  • The output is not just "concerns found"; it is a source-backed recommendation: proceed, flag, or reject.

Methodology validation agent

  • Prompt asks for arithmetic verification, text-table consistency checks, p-value logic, and detection of design flaws.
  • It can execute Python through Code Interpreter, with `python_repl` as fallback.
  • It also consumes methodology reporting output so CASP "Can't tell" findings can be reclassified as either reporting gaps or real flaws.
  • Faithfulness rules are unusually strict: only flag errors that can be verified from explicit manuscript text.
Validation prompt focus: 1. Verify arithmetic and denominators 2. Check text/table/figure consistency 3. Test p-value and CI logic 4. Flag only errors that can be explicitly verified

Validation prompt excerpt

ONLY flag errors you can verify from
explicit manuscript text.
NEVER infer values that are not stated.
Every error MUST have a direct quote.

Integrity prompt excerpt

When multiple authors share the same name:
1. Match affiliation
2. Match topic area
3. If still unclear, document ambiguity
4. Don't guess
Synthesis Layer

The last two agents mostly rewrite and prioritize prior machine judgments

Baseline synthesizer

Reads all prior agent JSON results from S3 using a fixed session-based key pattern and applies an editorial decision framework spanning scope, novelty, methodology, and ethics.

Writer

Receives the synthesizer's structured decision and is explicitly told not to re-evaluate the manuscript or introduce new findings.

Editorial implication

Most substantive judgment is upstream. By the time the commentary is written, the main leverage is in how earlier prompts framed evidence and thresholds.

Agent Main inputs Prompt style
Baseline synthesizer Scoping, ethics, novelty, integrity, reporting, validation JSON outputs Editorial policy synthesis with explicit decision options: Accept / Transfer / Reject / Further Investigation
Writer Final structured decision only Transformation prompt: turn structured fields into professional prose without changing the decision

Feedback Prompts

Questions to put in front of editors

Thresholds

  • Where should "transfer" end and "reject" begin?
  • How much novelty is enough for desk review progression?
  • Which ethics gaps are fixable versus fatal?

Authority

  • Which guidance sources should dominate when sources conflict?
  • Should BMJ policy outrank generic open science norms in borderline cases?
  • When is a local checklist too stale to trust?

Escalation

  • Which uncertainty patterns should force human editorial review?
  • What level of author-name ambiguity is acceptable?
  • Should unusual study designs trigger methodology specialist review by default?

The most valuable editorial feedback is likely to be about source authority, fatal-vs-fixable defects, and where the agents should stop pretending confidence.
