BMJ Article Fit

How the editorial agents are prompted, grounded, and constrained

A walkthrough of the prompting strategies and resource packs used across backend/agents/, designed to help editors understand what the agents are really leaning on.

Prompt decomposition Local guidance packs Search and retrieval tools Structured editorial outputs

Prepared from the current repository structure in backend/agents.

Why Editors Should Care

These agents are not one generic reviewer

What the system is doing

  • Breaking screening into narrower editorial questions: scope, novelty, ethics, integrity, reporting, statistical verification
  • Giving each sub-question a different prompt, tool set, and evidence base
  • Forcing agents to return structured fields rather than free-form opinions
  • Passing outputs forward so later agents synthesize prior assessments

What editorial feedback can improve

  • Whether the right evidence sources are treated as authoritative
  • Whether decision thresholds match real editorial practice
  • Whether prompts over-weight reporting versus scientific merit
  • Where transfer/reject logic needs tighter policy language
The key question is not "Is the model smart?" It is "Which instructions and which evidence sources are steering each editorial judgment?"

Prompting Pattern 1

Role-specialized prompts

Every agent is cast as a specific reviewer type with a clear mission. This focuses the model on one editorial question at a time.

Scoping Agent

You are a Scoping Specialist for {journal_name}.

Your task: Evaluate if a manuscript fits
{journal_name} based on BOTH topic relevance
AND study type.

Ethics Agent

You are an expert ethics compliance reviewer
for medical research manuscripts.

Novelty Agent

You are an editorial screening specialist
for BMJ journals. Your role is to evaluate
manuscript novelty, originality, and importance
by searching literature.

Integrity Agent

You are an expert editorial assistant
specializing in author integrity verification
for medical journal submissions.
Why this matters: A scoping specialist won't wander into ethics commentary. An integrity checker won't re-evaluate novelty. Each agent stays in its lane.
Prompting Pattern 2

Retrieval before judgment

Prompts explicitly name files, URLs, or search tools to consult before making decisions. The model gathers evidence first.

Scoping: Read local files

Use file_read to access:
- journal_scopes/{journal}/scope_summary.txt
- journal_scopes/{journal}/article_types.txt

Ethics: Crawl policy URLs

For EACH URL:
1. First try get_url_content(url)
2. If empty, fallback to web_extract
3. Extract ALL requirements, checklists,
   and criteria from retrieved content

Novelty: Search literature

Search literature using:
- OpenAlex for academic publications
- PubMed for biomedical literature
- Google Scholar and web sources
- Extract content from relevant papers

Integrity: Check author databases

Use MCP tools to gather information:
- search_pubpeer: Post-publication concerns
- validate_orcid: Author profiles
- search_openalex: Metrics and H-index
- search_pubmed: Publications

Each agent has access to different evidence sources suited to its question.

Prompting Pattern 3

Structured outputs

Agents emit structured decisions (Pydantic models) instead of free-form prose. This keeps downstream synthesis predictable.

Scoping output structure

Provide: - topic_fits_journal: Yes/No/Uncertain - study_type_fits_journal: Yes/No/Uncertain - decision: Keep/Transfer/Reject/Further Investigation - reasoning: Explain both assessments

Novelty output structure

Provide: 1. Literature volume summary 2. Key competing works (3-5 papers) 3. Novelty assessment: Level + evidence 4. Importance assessment 5. Rationale quality 6. Decision: Proceed/Revise/Reject
Why structured outputs matter: The synthesizer can programmatically combine decisions from 6+ agents. Free-form text would require re-interpretation.
Prompting Pattern 4

Learning from examples

Some agents include real examples of past decisions to calibrate their judgments. This helps the model understand what "Transfer" or "Reject" really means.

How it works

The scoping and novelty agents can load examples from JSON files:

few_shot_examples.json contains real manuscripts with: - title and abstract - decision made (Keep/Transfer/Reject) - reasoning provided - transfer target if applicable

These examples are appended to the prompt with: "Here are examples of previous evaluations to guide your assessments."

Example entry (scoping)

{ "title": "Effect of Mental Health Laws on ED Stay", "decision": { "topic_fits_emj": "No", "study_type_fits_emj": "Uncertain", "decision": "Transfer", "reasoning": "Primary focus is mental health policy, not emergency medicine processes." }, "transfer": { "target_journal": "mentalhealth", "reasoning": "BMJ Mental Health covers psychiatric research and policy." } }

Tool-constrained reasoning

Scoping Agent

Journal fit uses local scope packs

How the agent reads journal scope

The scoping agent uses file_read to access journal-specific files stored locally:

  • journal_scopes/{journal}/scope_summary.txt - What topics the journal covers
  • journal_scopes/{journal}/article_types.txt - What study types are accepted

The {journal} placeholder is replaced at runtime (e.g., "emj" becomes "Emergency Medicine Journal").

Decision criteria from the prompt

## Decision Rules Keep for {journal_name}: Both topic AND study type fit Transfer: Topic fits another BMJ journal OR study type fits another journal better Reject: Not related to human/population health outcomes (fundamental flaw) Further Investigation: Unclear if any BMJ journal is suitable

Full scoping prompt excerpt

You are a Scoping Specialist for {journal_name}.

Your task: Evaluate if a manuscript fits {journal_name} based on BOTH topic relevance AND study type.

## Available Tools
Use file_read to access:
- journal_scopes/{journal}/scope_summary.txt ({journal_name} topic scope)
- journal_scopes/{journal}/article_types.txt ({journal_name} accepted study types)

## Evaluation Criteria

### 1. Topic Fit (topic_fits_journal)
Read the scope_summary.txt file to understand what topics {journal_name} covers.
- Yes: Primary focus aligns with {journal_name}'s scope
- No: Primary focus belongs to a different specialty
- Uncertain: Mixed focus or unclear from abstract
Novelty Agent

Novelty uses live literature search

The four editorial questions

The novelty prompt frames specific questions editors need answered:

Answer these 4 critical editor questions: 1. How much and what research already exists? 2. Is the topic novel, interesting and important? 3. Is there a good rationale for the question? 4. What other research efforts already exist?

Search strategy instructions

Step 1: Recent Literature (2022-2025) Search last 3 years using key concepts. Focus on: question, population, intervention, outcomes. Step 2: Expand if Sparse If <10 papers found, expand to 7 years. Step 3: Landmark Studies Search highly-cited foundational work.

Novelty evaluation criteria from prompt

Novelty Level:
- Highly Novel: Unexplored question or groundbreaking approach; <3 similar studies
- Novel: Significant new angle, population, or methodology; meaningful advance
- Incremental: Minor additions; confirms prior findings in new setting
- Not Novel: Duplicates existing research

Existing Research Volume:
- Extensive: >50 highly relevant publications (last 3 years)
- Moderate: 10-50 relevant publications
- Limited: 3-10 relevant publications
- Minimal: <3 relevant publications
Ethics Agent - Part 1

Policy sources and compliance framework

The ethics agent retrieves guidance from multiple authoritative sources and builds compliance checklists from the crawled content.

Policy sources checked

  • UKRI/MRC: UK Research and Innovation ethics requirements for medical research
  • NHS HRA: Health Research Authority decision tools and REC approval criteria
  • BMJ policies: Journal-specific publication ethics and author requirements
  • Declaration of Helsinki: Fundamental principles for medical research on humans
  • ICMJE: International Committee of Medical Journal Editors guidelines
  • COS: Center for Open Science preregistration standards

How retrieval works

For EACH URL: 1. First try get_url_content(url) to retrieve cached content 2. If get_url_content returns empty, fallback to web_extract tool 3. Extract ALL requirements, checklists, and criteria from retrieved content

The repo contains pre-crawled content at backend/agents/ethics/crawled_data/ for faster access.

Ethics Agent - Part 2

Evaluation process and sub-agents

Main ethics compliance check

## Step 3: Evaluate Against Extracted Checklists For EACH criterion extracted, provide: - Criterion: [Requirement text from guideline] - Source: [Specific URL where found] - Evidence: Quote from manuscript or "NOT FOUND" - Compliance: LOW/MEDIUM/HIGH - Reasoning: Why this rating, citing guideline ## Step 4: Determine Overall Level - HIGH: All mandatory at HIGH, no LOW - MEDIUM: Most met, some at MEDIUM - LOW: Any mandatory criterion at LOW

Secondary ethics checks

After the main pass, additional agents may run:

  • Pre-specification agent: Reads Declaration of Helsinki via file_read, maps manuscript to relevant articles
  • Open science agent: Uses COS preregistration guidance, checks timeline consistency
  • Conflict-of-interest agent: Compares declared COIs with ORCID/publication history

These only run if the initial ethics pass is not LOW.

COI agent excerpt

Compare current declaration with ORCID and publication history.
Highlight both DECLARED and UNDECLARED conflicts. Check ALL authors.

Integrity Concerns levels:
- None: No concerns found
- Minor: Corrections or minor issues resolved
- Moderate: Unresolved concerns or multiple corrections
- Severe: Retractions, fraud allegations, or serious misconduct
Methodology Reporting - Part 1

Detecting the right reporting guideline

The methodology agent first determines which EQUATOR reporting guideline applies to the study type.

Study type classification

## Analysis Approach 1. Read Carefully: Focus on: - Methods section for study design - Abstract for study type/population - Results for data types and analysis 2. Key Indicators: - Humans vs Animals - Quantitative vs Qualitative - Randomized Trial (look for randomization, control groups) - Review/Meta-analysis - Observational (cohort, case-control) - Diagnostic Test - Prediction Model - Case Report

Guideline routing

Based on classification, the agent routes to:

  • CONSORT: Randomized controlled trials
  • PRISMA: Systematic reviews and meta-analyses
  • STROBE: Observational studies
  • TRIPOD: Prediction models
  • CARE: Case reports

Local flowchart.yaml stores an EQUATOR decision tree. If the study doesn't fit standard categories, the agent searches the EQUATOR Network.

EQUATOR fallback: "If the study design doesn't clearly fit standard categories, use web_search to search the EQUATOR Network for specialized reporting guidelines."
Methodology Reporting - Part 2

Parallel compliance assessment

Once the guideline is determined, three specialized sub-agents evaluate compliance in parallel.

Guidelines Specialist

Qualitative assessment: strengths, gaps, and methodology issues.

Checklist Auditor

Scores every checklist item. Uses calculate_score for quantitative compliance.

CASP Analyzer

Loads appropriate CASP checklist from local files for methodological appraisal.

Checklist auditor prompt excerpt

You are a Checklist Compliance Auditor. Evaluate EVERY item in the checklist.

For each item, determine compliance status:
- Fully Compliant (1.0): Explicitly addressed, specific details provided
- Partially Compliant (0.5): Mentioned but lacks specificity, incomplete
- Non-Compliant (0.0): Not mentioned, no evidence found

## Self-Critique Process
Before finalizing EACH item, ask yourself:
1. "Am I using the same evidence standard as previous items?"
2. "If I saw similar evidence for another item, would I score it the same?"
3. "Can I point to specific text that justifies this score?"

CASP checklists are stored locally at casp_checklists/randomised_control_trials, cohort_study, qualitative, etc.

Integrity Agent

Author verification is search-heavy

The integrity agent checks author history across multiple databases, with explicit instructions for handling ambiguity.

The three key questions

Your role is to answer these questions for each author: 1. Do the author(s) have any papers on PubPeer with unanswered integrity concerns? Have they had content retracted before? 2. Do the author(s) have ORCID/online presence, and have they published in this topic area? 3. Check integrity indicators: H-index, publication patterns, affiliation consistency

Disambiguation strategy

When multiple authors share the same name: 1. Match affiliation: Compare submitted affiliation with author profiles 2. Match topic area: Check if publications align with manuscript research field 3. If still unclear: Note as "Unable to verify - multiple matches" 4. Don't guess: Document ambiguity rather than making assumptions

Decision logic from prompt

Decision Logic:
- Proceed: No significant concerns found
- Flag for Review: Minor concerns or unable to verify some authors
- Reject: Severe integrity concerns (retractions, fraud, multiple red flags)

Be thorough but efficient. Focus on red flags and significant findings.
Methodology Validation Agent

Statistical verification is computation-heavy

The validation agent can execute Python code to verify calculations, and has strict rules about what counts as an error.

Verification tasks

Task 1: Statistical Verification (Python) - Arithmetic: "298 of 511 (58.32%)" execute: 298/511 * 100 - Totals: Subgroup sums = reported totals - Text-table consistency: Cross-reference ALL values across sections Task 2: Statistical Logic - P-value direction: "non-significant" requires P > 0.05 - CI coherence: If CI excludes null, result must be significant - Sample size: N at each stage matches flow diagram

Faithfulness rules

## CRITICAL - ONLY flag errors you can verify from explicit manuscript text - NEVER infer values that aren't stated - NEVER assume calculations the authors didn't report - If uncertain, mark confidence as "low" rather than fabricating errors - Every error MUST have a direct quote from the manuscript

This prevents the agent from inventing errors that don't exist in the manuscript.

CASP integration: Validation receives methodology reporting output. For each "Can't tell" item, it determines if it's a fundamental flaw or just a reporting gap.
Synthesis Layer

The last two agents rewrite and prioritize prior judgments

Baseline synthesizer

Reads all prior agent JSON results and applies an editorial decision framework spanning scope, novelty, methodology, and ethics.

Writer

Receives the synthesizer's structured decision and is explicitly told not to re-evaluate the manuscript or introduce new findings.

Editorial implication

Most substantive judgment is upstream. By the time the commentary is written, the main leverage is in how earlier prompts framed evidence.

Synthesizer excerpt

Review the complete assessment from six
specialized agents and make a final
editorial decision following BMJ's
editorial criteria.

Decision options:
- Accept
- Transfer
- Reject
- Further Investigation

Writer excerpt

You DO NOT re-evaluate the manuscript.
DO NOT add findings not in the input.
DO NOT change the decision.

Transform structured fields into
professional prose for editors.

Feedback Prompts

Questions to put in front of editors

Thresholds

  • Where should "transfer" end and "reject" begin?
  • How much novelty is enough for desk review progression?
  • Which ethics gaps are fixable versus fatal?

Authority

  • Which guidance sources should dominate when sources conflict?
  • Should BMJ policy outrank generic open science norms?
  • When is a local checklist too stale to trust?

Escalation

  • Which uncertainty patterns should force human review?
  • What level of author-name ambiguity is acceptable?
  • Should unusual study designs trigger specialist review by default?
The most valuable editorial feedback is likely to be about source authority, fatal-vs-fixable defects, and where the agents should stop pretending confidence.
